## 2. 业务背景

### 2.1 业务场景
TermAgent 是一款面向**所有需要使用命令行界面的用户**的桌面终端程序。它通过将传统终端与现代浏览器般的交互体验相结合，并逐步叠加AI能力，最终实现"让任何人都能轻松操作命令行"的目标。

### 2.2 业务目标
- **降低命令行使用门槛**：通过自然语言描述，让AI辅助生成和执行命令
- **提升终端操作效率**：简化复杂命令的编写，提供智能提示和自动化
- **统一跨平台体验**：一次开发，多平台发布，保持一致的交互体验

### 2.3 实施策略（分阶段）

| 阶段 | 目标 | 状态 |
|------|------|------|
| Phase 1 | 终端替代：稳定、高质量的基础终端程序 | **本期范围** |
| Phase 2 | AI对话界面：侧边AI对话框，支持对话式交互 | 规划中 |
| Phase 3 | AI自动化：命令白名单、用户授权执行 | 规划中 |

### 2.4 相关角色

| 角色 | 职责 | 权限级别 |
|------|------|----------|
| **最终用户** | 使用终端执行命令，与AI对话，授权AI执行命令 | 普通用户 |
| **系统管理员** | 配置AI服务、调整安全策略、管理白名单 | 管理员 |

## 3. 功能需求

### 3.1 会话管理

#### 功能描述
支持多标签页式会话管理，操作体验与现代浏览器一致。

#### 用户交互
- `Ctrl+T`：新建标签页
- `Ctrl+W`：关闭当前标签页
- `Ctrl+Tab`：切换到下一个标签页
- `Ctrl+Shift+Tab`：切换到上一个标签页
- 标签页支持拖拽排序

#### 业务规则
- 新建标签页时，自动在用户默认Shell中打开
- 关闭标签页时，如果Shell进程未退出，提示用户确认
- 退出程序时，保存当前所有标签页状态
- 重启程序时，恢复上次打开的标签页及工作目录

#### 异常处理
| 异常场景 | 处理逻辑 |
|----------|----------|
| Shell异常退出 | 在标签页显示退出码和错误信息，提供"重新打开"按钮 |
| 标签页进程未结束 | 弹出确认对话框："该标签页有进程正在运行，确定要关闭吗？" |

---

### 3.2 终端仿真

#### 功能描述
提供完整的终端仿真能力，兼容SSH、tmux、screen等传统协议和工具。

#### 用户交互
- 用户在终端区域输入命令并执行
- 支持所有标准终端操作（上下箭头查看历史、Tab补全等）

#### 业务规则
- **Shell选择**：优先使用用户配置的Shell路径；未配置时，自动检测系统默认Shell
  - Windows：优先 PowerShell/CMD，检测 Git Bash/WSL
  - macOS：检测 zsh/bash
  - Linux：检测 zsh/bash/dash
- **编码支持**：UTF-8，支持ANSI颜色转义序列
- **Unicode支持**：完整支持Unicode字符，包括emoji和双宽字符
- **终端类型**：xterm-256color

#### 数据要求
| 数据项 | 类型 | 说明 |
|--------|------|------|
| shell_path | string | 用户自定义Shell路径，可为空 |
| default_shell | string | 系统检测到的默认Shell路径 |

#### 异常处理
| 异常场景 | 处理逻辑 |
|----------|----------|
| 指定Shell不存在 | 回退到系统默认Shell，提示用户配置错误 |
| Shell启动失败 | 显示错误信息，提供重新选择Shell的选项 |

---

### 3.3 快捷键体系

#### 功能描述
沿用经典终端习惯，同时支持现代浏览器的导航快捷键。

#### 终端习惯（必须兼容）
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+C` | 中断当前进程 / 取消输入 |
| `Ctrl+D` | EOF（关闭Shell） |
| `Ctrl+Z` | 挂起当前进程 |
| `Ctrl+S` | 暂停输出 |
| `Ctrl+Q` | 恢复输出 |
| `Ctrl+L` | 清屏 |
| `Ctrl+R` | 搜索命令历史 |

#### 浏览器习惯（应该支持）
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+T` | 新建标签页 |
| `Ctrl+W` | 关闭标签页 |
| `Ctrl+Tab` | 切换到下一个标签页 |
| `Ctrl+Shift+Tab` | 切换到上一个标签页 |
| `Ctrl+Shift+C` | 复制选中内容 |
| `Ctrl+Shift+V` | 粘贴 |
| `Ctrl++/-` | 放大/缩小字体 |
| `Ctrl+0` | 重置字体大小 |

---

### 3.4 命令操作

#### 功能描述
支持完整的命令历史管理和回溯功能。

#### 用户交互
- **历史搜索**：`Ctrl+R` 打开反向增量搜索，输入关键字过滤历史命令
- **滚动回溯**：鼠标滚轮或 PageUp/PageDown 键浏览历史输出
- **复制粘贴**：选中文字后 Ctrl+Shift+C 复制；Ctrl+Shift+V 或鼠标右键粘贴

#### 业务规则
- 命令历史**本地持久化存储**，跨会话保留
- 历史记录包含：命令内容、 timestamp、执行目录、退出码
- 默认保留最近 10000 条历史记录（可配置）
- 支持按日期范围和命令内容搜索

#### 数据要求
| 数据项 | 类型 | 说明 |
|--------|------|------|
| command | string | 命令内容 |
| timestamp | datetime | 执行时间 |
| working_directory | string | 执行时所在目录 |
| exit_code | integer | 退出码 |

---

### 3.5 界面定制

#### 功能描述
提供主题切换和字体大小调整功能。

#### 用户交互
- **主题切换**：设置页面选择深色/浅色主题
- **字体调整**：`Ctrl++/-` 调整，`Ctrl+0` 重置
- **字体选择**：支持用户选择等宽字体（默认使用系统等宽字体）

#### 业务规则
- 主题设置**即时生效**，无需重启
- 字体设置**持久化保存**
- 记住用户最后一次的字体大小设置

---

### 3.6 AI 配置（Phase 2 预留）

#### 功能描述
存储AI服务配置，为Phase 2 AI对话界面提供基础设施。

#### 数据要求
| 数据项 | 类型 | 说明 |
|--------|------|------|
| api_endpoint | string | API地址（如 OpenAI Compatible） |
| api_key | string | API密钥（加密存储） |
| model | string | 模型名称 |
| base_url | string | 基础URL（可选） |

#### 业务规则
- **密钥存储**：API密钥通过本地配置文件加密存储
- **密钥管理**：用户可在设置页面查看、修改、删除API配置
- **安全性**：配置文件使用系统级加密存储（如 Windows Credential Manager / macOS Keychain / Linux libsecret）

---

### 3.7 命令执行授权（Phase 3 预留）

#### 功能描述
通过黑白名单机制控制AI执行命令的权限。

#### 业务规则
| 规则类型 | 说明 |
|----------|------|
| **白名单** | 匹配白名单的命令，AI可自动执行 |
| **黑名单** | 匹配黑名单的命令，禁止AI自动执行 |
| **默认行为** | 高风险命令默认在黑名单内，必须用户手动复制执行 |

#### 匹配规则
- 支持简单正则表达式匹配
- 示例：`rm -rf /.*` 可匹配所有强制删除根目录的命令
- 示例：`git .*` 可匹配所有git命令

#### 数据要求
| 数据项 | 类型 | 说明 |
|--------|------|------|
| pattern | string | 正则表达式模式 |
| action | string | "allow" 或 "deny" |
| description | string | 规则描述 |
| created_at | datetime | 创建时间 |

## 4. 数据实体

### 4.1 用户配置 (UserConfig)
| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| theme | string | 枚举: "dark", "light" | 界面主题 |
| font_size | integer | >= 8, <= 72 | 字体大小(px) |
| font_family | string | 可选 | 等宽字体名称 |
| shell_path | string | 可选 | 自定义Shell路径，为空则使用默认 |
| command_history_limit | integer | >= 100 | 历史记录保留条数 |

### 4.2 命令历史 (CommandHistory)
| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | uuid | 主键 | 唯一标识 |
| command | string | 必填 | 命令内容 |
| timestamp | datetime | 必填 | 执行时间 |
| working_directory | string | 必填 | 执行时工作目录 |
| exit_code | integer | 可选 | 退出码 |

### 4.3 会话状态 (SessionState)
| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | uuid | 主键 | 唯一标识 |
| tabs | array | 必填 | 标签页列表 |
| active_tab_index | integer | 必填 | 当前活动标签索引 |
| saved_at | datetime | 必填 | 保存时间 |

### 4.4 AI配置 (AIConfig)
| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | uuid | 主键 | 唯一标识 |
| name | string | 必填 | 配置名称（如 "OpenAI", "Claude"） |
| api_endpoint | string | 必填 | API端点地址 |
| api_key | string | 必填 | 加密存储的API密钥 |
| model | string | 必填 | 模型名称 |
| is_active | boolean | 默认false | 是否启用此配置 |

### 4.5 命令规则 (CommandRule)
| 属性 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | uuid | 主键 | 唯一标识 |
| pattern | string | 必填，正则表达式 | 匹配模式 |
| action | string | 枚举: "allow", "deny" | 执行动作 |
| description | string | 可选 | 规则描述 |
| priority | integer | 默认0 | 优先级（数字越大越优先） |
| created_at | datetime | 必填 | 创建时间 |

### 4.6 实体关系
```
UserConfig (1:1) ←→ SessionState (1:N) ←→ SessionTab (N:1) ShellProcess
AIConfig (1:N) → CommandRule (M:N) ←→ 执行决策
CommandHistory (N:1) → SessionTab
```

## 5. 业务流程

### 5.1 启动流程
```
用户启动程序
    ↓
加载用户配置 (UserConfig)
    ↓
恢复上次会话状态 (SessionState)
    ↓
对于每个保存的标签页：
    → 恢复工作目录
    → 启动Shell进程
    ↓
显示主界面（焦点在最后一个活动标签）
```

### 5.2 新建标签页流程
```
用户按 Ctrl+T 或点击"+"按钮
    ↓
创建新标签页
    ↓
获取Shell路径（用户配置 > 系统默认）
    ↓
启动Shell进程
    ↓
绑定输入/输出流
    ↓
标签页就绪，聚焦输入
```

### 5.3 关闭标签页流程
```
用户按 Ctrl+W 或点击关闭按钮
    ↓
检查Shell进程状态
    ↓
[进程仍在运行]
    → 显示确认对话框
    → 用户确认 → 发送SIGKILL → 关闭
    → 用户取消 → 取消操作
    ↓
[进程已结束]
    → 保存标签页状态到SessionState
    → 从界面移除标签页
    ↓
如果关闭的是最后一个标签页 → 退出程序
```

## 6. 非功能需求

### 6.1 性能要求
| 指标 | 要求 |
|------|------|
| 启动时间 | 相对同类竞品不过多占用即可 |
| 内存占用 | 空载时期望相对轻量，无具体限制 |
| CPU占用 | 空闲时CPU占用 < 1%，执行命令时合理 |

### 6.2 安全要求
| 维度 | 要求 |
|------|------|
| AI密钥存储 | 本地配置文件加密存储，使用系统级密钥链（Windows Credential Manager / macOS Keychain / Linux libsecret） |
| 配置文件保护 | 配置文件权限仅当前用户可读写 |
| 命令执行 | AI执行命令必须通过黑白名单授权 |

### 6.3 可用性要求
| 维度 | 要求 |
|------|------|
| 运行平台 | Windows 10+, macOS 11+, Ubuntu 20.04+ 或等价Linux发行版 |
| 维护窗口 | 无7x24强制要求，可维护性优先 |

### 6.4 扩展性要求
| 维度 | 要求 |
|------|------|
| AI服务 | 支持所有兼容OpenAI或Anthropic API格式的第三方服务 |
| 跨平台 | 一次编码，Windows/macOS/Linux三平台打包发布 |
