# TermAgent 技术规格说明书

## 1. 概述

### 1.1 项目简介

TermAgent 是一款面向所有需要使用命令行界面的用户的桌面终端程序，通过将传统终端与现代浏览器般的交互体验相结合，并逐步叠加 AI 能力，最终实现"让任何人都能轻松操作命令行"的目标。

### 1.2 技术栈

| 类别 | 技术选型 | 版本 |
|------|----------|------|
| 桌面框架 | Tauri | 2.x |
| 前端框架 | SolidJS | 1.9.x |
| 前端语言 | TypeScript | 5.x |
| 样式方案 | Tailwind CSS | 3.x |
| 终端仿真 | xterm.js | 5.x |
| 终端附加 | @xterm/addon-fit, @xterm/addon-web-links | latest |
| 构建工具 | Vite | 6.x |
| Rust MSRV | 1.76 | - |

## 2. 模块设计

### 2.1 会话管理模块 (TabBar)

#### 职责描述
- 管理多标签页式会话
- 提供标签页的创建、关闭、切换、拖拽排序
- 保存和恢复会话状态

#### 目录结构
```
components/TabBar/
├── TabBar.tsx          # 标签栏容器
├── TabItem.tsx         # 单个标签
└── NewTabButton.tsx    # 新建标签按钮
```

#### 核心接口

```typescript
// stores/sessionStore.ts
interface Tab {
  id: string;
  title: string;
  shellId: number;        // Tauri 侧 Shell 进程 ID
  workingDirectory: string;
  isActive: boolean;
  savedAt?: Date;
}

interface SessionState {
  tabs: Tab[];
  activeTabIndex: number;
}

// 创建新标签
function createTab(): Tab;
// 关闭标签
function closeTab(tabId: string): void;
// 切换标签
function setActiveTab(index: number): void;
// 拖拽重排序
function reorderTabs(fromIndex: number, toIndex: number): void;
```

#### 快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+T | 新建标签页 |
| Ctrl+W | 关闭当前标签页 |
| Ctrl+Tab | 切换到下一个标签页 |
| Ctrl+Shift+Tab | 切换到上一个标签页 |

#### 依赖模块
- shellService（启动/停止 Shell 进程）
- storageService（保存/恢复会话状态）

### 2.2 终端仿真模块 (Terminal)

#### 职责描述
- 基于 xterm.js 的终端渲染
- 处理用户输入，发送到 Shell 进程
- 解析和渲染 ANSI 转义序列
- 支持 Unicode 和双宽字符

#### 目录结构
```
components/Terminal/
├── Terminal.tsx        # 终端容器
├── TerminalSession.tsx # 单个终端会话
└── ANSIParser.ts       # ANSI 序列解析（可选）
```

#### 核心接口

```typescript
// services/shellService.ts
interface ShellSession {
  id: number;
  pid: number;
  workingDirectory: string;
}

// 创建 Shell 会话（Tauri 命令）
async function createShellSession(shellPath?: string): Promise<ShellSession>;

// 写入输入（Tauri 命令）
async function writeToShell(sessionId: number, data: string): Promise<void>;

// 读取输出（事件订阅）
subscribe('shell-output', (sessionId: number, data: string) => void);

// 发送信号（Tauri 命令）
async function sendSignal(sessionId: number, signal: 'SIGINT' | 'SIGTERM' | 'SIGKILL' | 'SIGTSTP' | 'SIGCONT'): Promise<void>;

// 关闭 Shell 会话（Tauri 命令）
async function closeShellSession(sessionId: number): Promise<void>;
```

#### 终端配置

```typescript
const terminalConfig = {
  cursorStyle: 'block',
  cursorBlink: true,
  fontSize: 14,
  fontFamily: 'monospace',
  theme: {
    background: '#1e1e1e',
    foreground: '#d4d4d4',
  },
  allowTransparency: true,
  scrollback: 10000,
};
```

#### 快捷键（终端习惯）

| 快捷键 | 功能 |
|--------|------|
| Ctrl+C | 中断当前进程 / 取消输入 |
| Ctrl+D | EOF（关闭 Shell） |
| Ctrl+Z | 挂起当前进程 |
| Ctrl+S | 暂停输出 |
| Ctrl+Q | 恢复输出 |

### 2.3 命令历史模块 (History)

#### 职责描述
- 本地持久化存储命令历史
- 支持按日期范围和命令内容搜索
- 跨会话保留历史记录

#### 目录结构

```
services/
└── historyService.ts    # 命令历史服务
```

#### 数据模型

```typescript
interface CommandHistory {
  id: string;              // UUID
  command: string;         // 命令内容
  timestamp: Date;         // 执行时间
  workingDirectory: string; // 执行时所在目录
  exitCode?: number;       // 退出码
}
```

#### 核心接口

```typescript
// 添加历史记录
async function addHistory(entry: Omit<CommandHistory, 'id'>): Promise<CommandHistory>;

// 搜索历史
async function searchHistory(query: {
  keyword?: string;
  fromDate?: Date;
  toDate?: Date;
  limit?: number;
}): Promise<CommandHistory[]>;

// 获取最近 N 条
async function getRecentHistory(limit: number): Promise<CommandHistory[]>;

// 清空历史
async function clearHistory(): Promise<void>;
```

#### 存储格式

历史记录存储在 `{app_data_dir}/history.json`，格式：

```json
{
  "commands": [
    {
      "id": "uuid",
      "command": "ls -la",
      "timestamp": "2026-03-19T10:30:00Z",
      "workingDirectory": "/home/user",
      "exitCode": 0
    }
  ],
  "version": 1
}
```

#### 依赖模块
- storageService（通用持久化）

## 3. 启动流程

```
用户启动程序
    ↓
加载 Rust 运行时
    ↓
Tauri 主线程初始化
    ↓
前端入口加载（SolidJS + Vite Dev Server 或 bundled）
    ↓
前端：从 Tauri 读取用户配置 (getConfig)
    ↓
前端：从 Tauri 恢复会话状态 (readConfig: session.json)
    ↓
对于每个保存的标签页：
    → 调用 create_shell（恢复工作目录）
    → 建立输出事件订阅
    ↓
渲染 TabBar + 活动 Terminal
    ↓
聚焦活动标签的输入
```

## 4. 新建标签页流程

```
用户按 Ctrl+T 或点击"+"按钮
    ↓
前端：sessionStore.createTab()
    ↓
调用 Tauri create_shell 命令
    ↓
Rust：检测 Shell 路径
    ↓
Rust：创建 PTY 进程
    ↓
Rust：返回 session_id 和 initial_output
    ↓
前端：创建 Terminal 组件
    ↓
前端：订阅 shell-output 事件
    ↓
渲染标签页，聚焦输入
```

---

## 5. 关闭标签页流程

```
用户按 Ctrl+W 或点击关闭按钮
    ↓
前端：检查 Shell 进程状态
    ↓
[进程仍在运行]
    → 显示确认对话框
    → 用户确认 → 调用 close_shell（SIGKILL）
    → 用户取消 → 取消操作
    ↓
[进程已结束]
    → 更新 sessionStore
    → 如果关闭的是最后一个标签页 → 调用 Tauri 退出
    ↓
保存会话状态到 session.json
```

## 6. 技术决策

### 6.1 终端渲染方案

- **选择**：xterm.js + @xterm/addon-fit + @xterm/addon-web-links
- **理由**：业界标准，支持 256 色、Unicode、链接检测，与 Tauri 配合良好

### 6.2 PTY 方案（Rust）

- **选择**：portable-pty
- **理由**：跨平台支持完善，API 简洁，与 tokio 配合良好

### 6.3 状态管理方案

- **选择**：SolidJS 原生 createSignal / createStore
- **理由**：SolidJS 响应式系统强大，无需额外依赖，性能最优

### 6.4 样式方案

- **选择**：Tailwind CSS
- **理由**：原子化 CSS 开发效率高，配合 dark/light theme 便捷

### 6.5 进程间通信

- **选择**：Tauri 命令 + 事件订阅
- **理由**：Tauri 原生支持异步命令和事件推送，适合 PTY 的实时输出场景

### 6.6 密钥存储

- **选择**：windows-sys + Credential Manager（Windows）
- **理由**：Phase 2 预留，系统级安全存储

## 7. 安全措施

### 7.1 配置文件保护

- 配置文件权限：仅当前用户可读写
- 实现：Rust 层写入时设置文件权限（Unix: 0o600, Windows: ACL）

### 7.2 API 密钥存储（Phase 2）

- 使用系统级密钥链
- Windows: Credential Manager
- macOS: Keychain
- Linux: libsecret

### 7.3 Shell 执行

- 所有 Shell 命令通过 PTY 执行，无法直接执行任意命令
- AI 执行命令必须通过黑白名单授权（Phase 3）
